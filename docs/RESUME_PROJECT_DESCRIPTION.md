# MedicalGraph - 医疗知识图谱问答系统

## 简历项目描述 (STAR法则)

### Situation（背景）
面对通用知识图谱系统在医疗领域的局限性，需要构建一个专业的医疗知识图谱问答系统，支持医疗实体识别、关系抽取、智能问答等核心能力，为用户提供准确、专业的医疗信息服务。

### Task（任务）
作为技术负责人，负责：
- 架构设计与技术选型
- 医疗知识融合引擎开发
- 智能问答系统实现
- 前端可视化组件开发
- 系统性能优化与测试

### Action（行动）
1. **架构设计**：设计分层架构，整合 FastAPI、Neo4j、LangChain/LangGraph 等技术栈
2. **知识融合引擎**：实现 EntityDisambiguator（实体消歧）、RelationAligner（关系对齐）、KnowledgeFusionEngine（知识融合引擎），支持 ICD-10 和 UMLS 医疗编码标准
3. **智能问答**：开发混合检索机制（向量检索 + 图检索），实现基于规则和LLM的医疗意图识别
4. **图谱可视化**：开发交互式图谱组件，支持节点悬停高亮、右键菜单、节点聚焦等功能
5. **供应商迁移**：将 LLM 供应商从 OpenAI 迁移到 DashScope，优化成本和响应速度
6. **性能优化**：引入缓存机制，将平均响应时间从 6.71s 优化至 3.14s，吞吐量提升 113%

### Result（结果）
- **功能完整性**：成功实现医疗实体识别（疾病、症状、药物、检查、治疗、解剖部位、科室）7大类型
- **意图识别准确率**：从 75% 提升至 87.5%
- **响应速度**：平均响应时间从 6.71s 优化至 3.14s，提升 53%
- **系统稳定性**：通过全链路测试，错误率低于 1%
- **技术文档**：完成项目转型文档、架构评审文档、优化报告等核心文档

### 核心技术亮点
- 知识融合引擎支持医疗领域标准编码映射
- 混合检索机制提升问答准确性
- 交互式图谱可视化提升用户体验
- 智能缓存策略优化系统性能

### 关键技术栈
- **后端**: FastAPI, Neo4j, LangChain, LangGraph, DashScope
- **前端**: React 19, TypeScript, Tailwind CSS, Vis Network
- **部署**: Docker, CI/CD
